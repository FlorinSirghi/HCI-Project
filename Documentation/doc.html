1. Abstract

This document presents the decisions and patterns of our UI design of the AiCo project (Airsoft Companion) which is an web (smartphone and desktop), smart watch and AR app representing a companion for Airsoft and is aimed as a documentation for the various functions of the app.

2. Introduction

2.1 Purpose

ModernSoft is an web app that aims to help users to browse, search, and discover new content from popular streaming platforms such as Netflix or Disney in a more personalized and informed way, made by team ModernSoft. The project's scope is to deepen our knowledge of UI/UX design and its prerequisites like conducting research tacatics, creating personas and creating storyboards, user flows and sketches and also to fulfill the requirement of the Human-Computer Interaction class.

2.2 Document conventions

This document follows the Scholarly HTML format, which is rich document format built entirely on open standards that enables the interoperable exchange of scholarly articles.

2.3 Intended Audience and Reading Suggestions

This document is made as a requirement for the first part of our project for the Human-Computer Interaction course and is intended for the evaluator teacher to help them navigate our app easier and to get a better understanding of the overall architecture of the app.

2.4 Product Scope

This project is made for the Human-Computer Interaction course taught at the Faculty of Computer Science, University A.I.Cuza in Iasi, Romania.

3. Overall Description

3.1 Product Perspective

The product represents our design of a web (smartphone, desktop), smart watch, AR app for the AiCo(Airsoft Companion) project which represents a companion for the game Airsoft the design is focused on the ease of use as Airsoft is a dynamic game

3.2 Product Functions

The main functions of the product are :

  • the ability to manage (create, join) a lobby to play Airsoft
  • the ability to find people to play Airsoft with
  • the ability to make use of the various in-game functions of the app such as : the map, the radar in order to win the game
  • the ability to report users for bad behaviour prior, during and after the game
  • the ability to check various statistics about games and other players

3.3 Operating Environment

The implementation of our design project is supposed to run on one side as a web app in the browser, this part of the app will include functionalities like finding people to play with or checking statistics, and on the other side as a app for smart watch and an AR headset which will be used mostly during the game.

3.4 Assumptions

The design is made under the asumption that any person, no matter their age or level of technology should be able to use the app easily and have fun while playing the game.

4. Interface

4.1 User Interfaces

The app for smart watch contains multiple screens :

  • Create a Lobby : represented by a form with fields for the lobby name, game mode, match duration and number of players in each team.
  • Join a Lobby : a simple form with only one field : the lobby password
  • Settings : to set/modify things like font-size, language, colors
  • Menu : a main page to jump around the various screens
  • Map : a map of the playground, contains red dot icons if the player spots and enemy and also green dot icons which show allies positions
  • Stats : a list of information based on previous matches like : number of game wins, number of wins, win percentage

The web app also contains various pages, besides those that are also present in the app for smart watch :

  • Find people to play with : a list of players that are close to you
  • Login
  • Create an account
  • Chat

The layout for the AR app contains various information about the game and player state such as : health left, ammo, time-left, score, players left in each team - shown using human-like icons.

4.2 Hardware Interfaces

The web app is made for a desktop browser, but can also be accessed from a smart phone using the browser.

5. System features

5.1 Managing a lobby

Our app provides the ability for the user to create or join a lobby to play Airsoft. In a lobby the players can set different settings such as : the game type, the number of players in each team, the duration of the match and the score required to win the game.

5.2 Finding people to play with

Another functionality that our app provides is the ability to find other people interested in playing airsoft. This can be done using the web app.

5.3 Use various in-game features in order to win

The application also provides in-game functionalities like a map for players to be able to find their way around the playing zone, a radar for situations when they are not 100% sure they have seen someone and other information such as the ammo left in their guns or information about the game such as rounds left, allies alive, enemies killed, etc.

5.4 Reporting users for bad behaviour prior, during and after the game

Users can also report other players for bad behaviour before, during and after the game, these reports show up in the admin panel to be processed and verified, leaving it up to the admin to decide if the report is valid and apply penalties.

5.5 Check various statistics about games and other players

By using our app, users can also see various statistics regarding the the games they've played and the teammates and enemies they played with. They can also see statistics about themselves like the number of games they won or the number of kills they got.
